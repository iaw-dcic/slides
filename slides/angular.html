¿Qué es AngularJS?

AngularJS es un framework estructural para la aplicaciones web dinámicas.

Permite utilizar HTML para construir templates y extender la sintaxis para expresar componentes específicos de manera clara y concisa.

Además, permite el enlace de datos y la inyección de dependencias, eliminando la necesidad de escribir código repetitivo.

Todo ocurre en el navegador, permitiendo cualquier tecnologia del lado del servidor.

Construcciones HTML

Angular permite nuevas construcciones utilizando directivas:

  • Vinculación de datos,
  • Estructuras de control para DOM, para repetir, mostra y ocultar fragmentos del DOM,
  • Soporte para formularios y validación,
  • Agregar nuevo comportamiento a elementos del DOM, como manejo de eventos,
  • Agrupar HTML en componentes reutilizables.

Modelo - Vista

Directivas

Son marcas en los elementos del DOM que le indica a AngularJS que le tiene que agregar un comportamiento específico.

Ejemplos: ng-app, ng-model, ng-bind

Cuando se inicia la aplicación, un compilador busca las directivas y las procesa, para lograr el comportamiento deseado.

Se pueden crear directivas propias.

Expresiones

Son fragmentos de código JavaScript que se colocan generalmente con el siguiente formato: {{expresión}}.

Por ejemplo:

  • 1+2
  • a+b
  • user.name
  • items[index]
<span>
  1+2={{1+2}}
</span>

Filtros

Los filtros pueden aplicarse a las expresiones en plantillas de vista con la siguiente sintaxis:

{{ expresion | filtro }}

También pueden encadenarse: 

{{ expresion | filtro1 | filtro2 }}

y pueden tener argumentos: 

{{ expresion | filtro:argumento1:argumento2 }}

Controladores

Los controladores proporcionan la lógica de negocios de la aplicación.

Un controlador es una objecto javascript que tiene un alcance (scope), el cual será utilizado para comunicarse con la vista

Usar Controladores para:

  • Inicializar el alcance (scope)
  • agregar comportamiento para el alcance (scope)

No usarlos para:

  • Manipular DOM (usar databinding y directivas)
  • Formatear entrada (usar controles de formularios)
  • Filtrar salida (usar filtros)
  • Compartir código (usar servicios)

Controladores

var controladorPedido = angular.module('controladorPedido', []);

controladorPedido.controller('PedidoPizzas', function() {
  var pedido = this;
  pedido.pizzas = [];
  angular.forEach(jsonPizzas, function(pizza) {
    pizzaEnPedido = jQuery.extend({}, pizza);
    pizzaEnPedido.cantidad = 0;
    pedido.pizzas.push(pizzaEnPedido);
  });

  pedido.total = function() {
    var total = 0;
    angular.forEach(pedido.pizzas, function(pizza) {
      total += pizza.cantidad * pizza.precio;
    });
    return total;
  };
});

Servicios

La lógica de negocios de vistas independientes no tiene que estar en un controlador. Dicha lógica debe estar en un servicio.

Los controladores son específicos de una vista, mientras que los servicios son específicos de una aplicación.

Los controladores son responsables de vincular el modelo a la vista. Los modelos puede ser obtenidos de un servicio.

Los controladores no son responsables de manipular (crear, destruir, actualizar) los datos. En cambio, los servicios son los responsables.

Servicios

var miApp = angular.module('miApp', []);

miApp.controller('ViewCtrl', function($scope, servicio) {
  $scope.datos = servicio.datos;
});

miApp.controller('ModifyCtrl', function($scope, servicio) {
  $scope.datos = servicio.datos;
});

miApp.factory('servicio', function() {
  var factory = {};
  factory.datos = [{nombre: 'Mariano'}, {nombre: 'Fernando'}, {nombre: 'Diego'}];
  return factory;
});

Modulos

Se puede pensar en un módulo como un contenedor para las diferentes partes de su aplicación - controladores, servicios, filtros, directivas, etc.

Angular no tiene un método principal donde todo se inicializa. Los módulos proveen una forma declarativa de especificar como una aplicación debe inicializarse.

  var miApp = angular.module('miApp', []);

  miApp.filter('bienvenido', function() {
    return function(nombre) {
      return 'Hola, ' + nombre + '!';
    }

  });'

Rutas

Todo ocurre en una única página. ¿Qué hace el botón atras?

var pizzasApp = angular.module('pizzasApp', ['ngRoute', 'controladorPedido']);

pizzasApp.config(['$routeProvider',
  function($routeProvider) {
    $routeProvider.
      when('/pedido', {
        templateUrl: 'views/pedido.html',
        controller: 'PedidoPizzas'
      }).
      otherwise({
        redirectTo: '/pedido'
      });
  }]);

Conclusión

AngularJS es un framework modular de javascript para desarrollar lo que se denomina SPA (Single Page Applications)

Tiene muchas característias interesantes:

  • Desacopla la manipulación DOM de aplicación lógica. Esto mejora dramáticamente la capacidad de prueba del código.
  • Desacopla el lado del cliente de una aplicación desde el lado del servidor. Esto permite que el trabajo de desarrollo para avanzar en paralelo, y permite la reutilización de ambos lados.
  • Facilita el testing de la aplicación.
  • Permite ser utilizado con otras librerías de JS (como jQuery)
