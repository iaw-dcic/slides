Doctype

  • La declaración de tipo de documento (DOCTYPE) para HTML5 es:
  • 
<!DOCTYPE html>
  • La declaración del tipo de codificación de los caracteres se puede realizar de la siguiente manera:
  • 
<meta charset="UTF-8">

Elementos nuevos en HTML5

Los más interesantes son:

  • elementos semánticos como <header>, <footer>, <article>, y <section>
  • atributos para input como number, date, time, calendar, y range.
  • elementos gráficos: <svg> y <canvas>.
  • elementos multimedia: <audio> y <video>.

Drag and Drop

function allowDrop(ev) {
    ev.preventDefault();
}
function drag(ev) {
    ev.dataTransfer.setData("text", ev.target.id);
}
function drop(ev) {
    ev.preventDefault();
    var data = ev.dataTransfer.getData("text");
    ev.target.appendChild(document.getElementById(data));
}
<div id="div1" ondrop="drop(event)" ondragover="allowDrop(event)"></div>

Web Workers

Cuenta: 

var w;
function startWorker() {
  if(typeof(Worker) !== "undefined") {
    if(typeof(w) == "undefined") {
      w = new Worker("worker.js");
    }
    w.onmessage = function(event) {
      document.getElementById("result").innerHTML = event.data;
    };
  } else {
    document.getElementById("result").innerHTML = 
    "Sorry, your browser does not support Web Workers...";
  }
}
function stopWorker() { 
  w.terminate();
  w = undefined;
}

Otros Ejemplos

HTML5 Server-Sent Events

  • Un server-sent event es cuando una página web se actualiza automáticamente a partir de informacion desde el servidor
  • Esto se podía realizar anteriormente, pero a través de pooling desde el cliente, no iniciado por el servidor

Application Cache

  • Permite que una aplicación web sea almacenada en cache en el cliente y accesible sin conexión a internet.
  • Provee las siguientes ventajas:
    • Navegación offline, los usuarios puede usar la app sin conexión a internet.
    • Velocidad, los recursos en cache se cargan más rápidamente.
    • Disminuye la carga en el servidor, solo se descargan los recursos actualizados o cambiados.

Otros Ejemplos

Web Sockets

  • Web Sockets permite una comunicación bidireccional (de bajo nivel) para aplicaciones web entre el cliente y servidor.
  • Una vez que se obtiene una conexión con un servidor web, el cliente puede:
    • enviar un mensaje usando el método send()
    • recibir mensajes a través del evento onmessage
    • detectar cambios de estado en la conexión, tales como la conexión inicial, o la interrupción de la misma.
